# Internal reminder email workflow — La Carolina

Use this reference when preparing reminder emails for La Carolina internal meetings, action items, or committee follow-ups.

## Recipient grouping rule

- Send **one email per person** whenever possible.
- If a person has individual tasks plus shared tasks, include all of them in that person's single email.
- For shared tasks, explicitly say who the person must coordinate with: `Este tema debe coordinarse con X y Y`.
- Do not create duplicate emails just because the same person appears in multiple rows/groups.

## Department/team mapping learned from reminders

- Yisseth handles **Psicología** and **Comunicaciones**: `psicologia@lacarolina.com.co`.
- Víctor Sandoval / Equipo IT: `desarrollo@lacarolina.com.co`.
- If the source list says **Diseñador** but the user says there is no designer, assign that communications/design-related task to **Yisseth / Comunicaciones** instead.

## Attachment/note pattern

When the user provides an Excel or document to include in specific reminders:

1. Mention the file in the affected recipients' email under `Adjunto` or in a paragraph before the closing.
2. If the file is for Edith Royero and John Guerrero with departmental savings presentation context, use this exact note or close variant:

> Adicionalmente, se te anexa el Excel de presentación de ahorros para los departamentos, para tu revisión y para enviarlo lo antes posible a los directores de departamentos.

3. Attach only to the specified recipients, not to every reminder.

## Cross-check before drafting follow-up reminders

When Liliana asks for pendientes from the latest La Carolina leaders/directives meeting and says someone responded, do not rely only on the Obsidian pending list. Cross-check:

1. The main Obsidian meeting note under `empresas/la-carolina/Reunion De Lideres/`.
2. The immediately previous leaders/directives meeting note in the same folder, unless Liliana names a different prior meeting. She expects follow-up drafts to combine the current meeting plus prior-meeting items that are still not closed, while excluding items already closed/responded.
3. Any saved email attachments in that meeting's `adjuntos/` folder.
4. Corporate Outlook replies to the reminder threads, especially subjects like `Pendientes para reunión...`, `Comité Directivo`, `plan de ahorros`, and sender addresses for the department leads.

Classify results explicitly:

- `Cumplido / respondido`: the person replied with a concrete answer or the deliverable was presented (example: Edith Royero answered Finanzas; departmental savings presentation was delivered).
- `Parcial / por validar`: the person replied with materials but missing figures, indicators, evidence, or close dates (example: Psicología/Comunicaciones response with scripts/HTML/encuesta but no savings metrics).
- `Abierto / requiere seguimiento`: no substantive response or no evidence of completion.

For follow-up emails, draft by department/person and reference only what remains open for that area. If one area did comply, acknowledge it in that area's draft and ask only for support/validation still missing rather than sending the same generic reminder again.

## Draft style

- Tone: formal, direct, practical, without over-explaining.
- Subject pattern: `Recordatorio de compromisos asignados – Comité Directivo [fecha]` or `Seguimiento pendientes [área] — reunión líderes La Carolina [fecha]`.
- Body structure:
  - Greeting.
  - Short context sentence referencing the meeting.
  - Bullet list of assigned commitments.
  - For shared items, coordination note in the same bullet.
  - If prior response exists, begin with a brief acknowledgment and then list only validation/support gaps.
  - Attachment note if applicable.
  - Closing: `Quedo atenta a cualquier novedad. Cordialmente, Liliana Anaya`

## Send-before-action rule

If the user asks to send the emails, confirm that sender, recipients, attachments, and missing emails are clear. If the user gives an explicit command such as `envia los correos desde lanaya@lacarolina.com.co`, proceed with the confirmed recipients and report only verified sends.
