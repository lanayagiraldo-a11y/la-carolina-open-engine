# Naming and user-facing positioning

Session learning: Liliana questioned why the financial framework was being framed as a "skill/agente". For this class of work, describe the system as a **metodología financiera de trabajo para Hermi**, not as autonomous agents.

Guidance:
- User-facing language: prefer "metodología financiera", "marco de análisis", "flujo de trabajo" or "skill de trabajo".
- Avoid saying that we created active/independent agents unless we truly configured autonomous processes.
- The 7 role files are references/perspectives for analysis, not bots that run on their own.
- Clarify briefly when needed: "queda como una metodología reutilizable que Hermi carga cuando me pidas análisis financieros; no actúa sola".
- If renaming or presenting deliverables, use a friendlier label such as `metodologia-financiera-la-carolina` in explanations, while preserving internal paths unless the user explicitly asks to rename files.
